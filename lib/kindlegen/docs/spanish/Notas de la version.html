Kindlegen versión 2.8, compilación 0208-797bf75

	   - Support for converting XMDF file format has been added. Refer Kindle Publishing Guidelines for more info.
	   - Support for epub3 toc nav element has been added. This is equivalent to the navMap element in the NCX (also known as NCX toc). Epub3 toc nav element is preferred over NCX navMap element, if both are given. But if the Nav HTML is not specified according to the specification, we will try to fall back to NCX if one exists. If we cannot fallback to NCX, then we would error out. Refer here for more info about EPUB3 toc nav specification.
	   - Support for epub3 landmarks nav element has been added. This is equivalent to the OPF guide element. Epub3 landmarks nav element is preferred over OPF guide element, if both are given. Refer here for more info about EPUB3 landmarks nav specification.
	   - Support for specifying the following metadatas in epub3 format
	       - Specifying cover image using properties attribute in manifest
	       - Specifying book level layout with “rendition:layout” property
	       - Specifying book level orientation with “rendition:orientation” property
	       - Specifying “dc:identifier” value in URN format.
	       - Refining authors in “dc:creator” metadata using marc relators scheme
	       - Providing multiple titles with one among them refined as main title using title-type property.
	     Refer here for more info.
	   - Support for referring images in NCX toc if and only if the same image is directly referred in the spine.
	   - Support for Galician language(gl, gl-es)

Kindlegen versión 2.7, compilación 1019-d6e4792

	   - Las URI de datos admiten imágenes y fuentes incrustadas para poder utilizarlas directamente como referencias en archivos  HTML y CSS (Si necesitas más información de las URI de datos, consulta http://dataurl.net).
	   - Se permite la representación en vertical en japonés, y diferentes modo de escritura en las páginas.  El valor puede especificarse en los archivos OPF mediante etiquetas meta (p. ej: <meta name=”primary-writing-mode” content=”vertical-rl” />).  Los valores admitidos son horizontal-lr, vertical-lr, horizontal-rl, vertical-rl.
	   - Se permite utilizar páginas enfrentadas y extensiones a doble página. El valor puede especificarse en OPF mediante el atributo “properties” de la etiqueta itemref. Los valores admitidos son facing-page-left, facing-page-right, page-spread-left, page-spread-right.
	   - Se ha añadido compatibilidad de Mobi7 para etiquetas Html con varias clases (p. ej: class="class1 class2 class3")
	   - Kindlegen da un mensaje de error cuando no se especifican los metadatos de la resolución original en el formato "número entero x número entero" en libros de formato fijo V2.
	   - Se valida si los ordinales usados en JSON de ampliación de una región son exclusivos en los libros de formato fijo V2.
	   - Se han añadido estadísticas de tamaño de archivos KF8 y M7 para mostrarlas después de la conversión
	   - Se ha mejorado el mensaje de error que aparece cuando falla la generación de la tabla de contenido debido a que las entradas de NCX apuntan a etiquetas con la propiedad display:none.  
	   - Se ha resuelto el problema relacionado con el manejo de etiquetas ancla anidadas en HTML. 
	   - Se manejan de forma adecuada las etiqueta section, aside, footer, figure, figcaption y article.

Kindlegen versión 2.5, compilación 0626-3a91e28

	   - Mejora de la conversión transparente PNG: imágenes PNG transparentes (ahora convertido a JPEG) usará el fondo blanco. Esta usado originalmente fondo negro durante la conversión causando experiencia de lectura degradado.
	   - Roturas intencionales de línea en Mobi7 al utilizar algunas de las propiedades CSS ("margin-top: 0px" y “margin-bottom: 0px”) se ha solucionado.
	   - Resueltas las cuestiones de sangría de Mobi7 relacionados con las balas y las listas numeradas. Esto se debió al uso de "margin-left", propiedad de CSS con balas y listas numeradas. En Mobi7, "margin-left", la propiedad está siendo ignorado por las balas y las listas numeradas. El mismo se seguirá conservado en KF8.
	   - Se han corregido problemas en la creación de libros Mobi7 con índices (por ej Diccionarios)
	   - Propiedad CSS "display: none" ya no induce un espacio adicional en Mobi7 para etiquetas insertadas.
	   - Las fuentes incrustadas de tipo PostScript/Type1 no se ve bien en los lectores Kindle eInk con KF8 apoyo. Kindlegen advierte sobre el uso de estas fuentes en el contenido. Kindlegen también hace la experiencia del usuario de las fuentes del sistema en los libros de este tipo (específicamente para dispositivos eInk) por defecto.
	   - Kindlegen graciosamente pasa por alto el documento EPUB3 de navegación (se especifica el uso de propiedades = "nav" en el manifiesto) y no añadirlos a la orden de lectura del libro de forma predeterminada.
	   - Resuelto el fracaso de conversión cuando dc: etiqueta de idioma en OPF tiene atributos (por ejemplo: atributo id).
	   - Problema resuelto en la construcción lógica de TOC, debido a las etiquetas de texto vacíos o de cierre automático etiquetas de texto en el archivo de NCX.
	   - Problema resuelto en la eliminación de las propiedades de CSS que no son compatibles con Kindle (por ejemplo: la altura de línea en píxeles/pt unidades). Kindlegen quitado la final "}" en un caso extremo específico cuando esta era la última propiedad y no tiene un punto y coma al final. El problema se soluciona en Kindlegen 2.5.
	   - Actualización opción “–releasenotes” en Kindlegen para señalar a los usuarios a notas de la versión detalladas alojados en la web en https://kindlegen.s3.amazonaws.com/manual.html.
